## 1．总目标与量化指标（开题与结题共同口径）

### 1.1 主目标（必达）

- [ ] 在 RailSem19 上完成至少 2 个语义分割模型的可复现实验：  
  - [ ] 1 个“精度优先基线”（例如 DeepLabV3+／SegFormer-B0）。  
  - [ ] 1 个“实时优先基线”（例如 BiSeNetV2／STDC-Seg／Fast-SCNN）。  
- [ ] 输出对比表：mIoU、mAcc、Params、FLOPs、端侧延迟（ms）或 FPS。  
- [ ] 论文中给出清晰的实验设置（输入分辨率、增强、学习率策略、评测协议、测速协议）。  

### 1.2 轻量化与部署目标（至少达成一条）

- [ ] 轻量化：参数量或 FLOPs 降低（建议 ≥30%），同时 mIoU 降幅可控（建议 ≤2 个百分点）。  
- [ ] 边缘部署：至少完成 FP16（ONNX → TensorRT engine），并给出端侧测速结果与可视化 Demo。  
- [ ]（加分项）INT8：完成 PTQ 校准或 QAT，并评估精度与速度变化。

### 4.2 数据预处理（务必可复现）

- [ ] 解析标注格式：  
  - [ ] 确认 GT 是单通道 label map（每像素类别 id）。  
  - [ ] 统一 ignore index（例如 255）。  
- [ ] 生成索引文件：  
  - [ ] `train.txt`、`val.txt`、`test.txt`（每行一张图路径与 mask 路径）。  
- [ ] 制作可视化检查脚本：  
  - [ ] 随机抽样 50 张叠加可视化，检查错位、label 值异常。  
- [ ] 数据增强配置固化（写进 config）：

- [ ] 输入尺寸：至少固定 1 个主尺寸（例如 512×512 或 1024×512）。  
- [ ] 评测频率：每 N iter 或每个 epoch 评一次，并保存 best。  
- [ ] 类别不平衡处理策略（可选）：加权 CE、Focal、Dice 等。
